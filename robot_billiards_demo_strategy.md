# 로봇팔 기반 당구 데모 전략 정리

## 0. 프로젝트 핵심 포지션

본 프로젝트의 핵심은 단순히 로봇팔로 공을 맞히는 것이 아니라, **범용 6축 로봇팔인 Indy7을 이용해 불완전한 미니 당구대 환경에서 경로를 탐색하고, 실제 타격 데이터를 통해 sim-to-real gap을 줄이며, 트릭샷 성공률을 높이는 시스템**을 보여주는 것이다.

핵심 주장:

> 기존 당구 로봇이 주로 정밀한 환경에서 포켓볼을 성공시키는 데 집중했다면, 본 프로젝트는 저가형 미니 당구대의 불완전한 물리 특성을 로봇이 반복 타격을 통해 보정하고, Indy7의 실제 실행 가능성을 고려하여 트릭샷 경로를 선택하는 자가 보정형 로봇팔 당구 시스템을 목표로 한다.

짧은 버전:

> 우리는 공을 잘 치는 로봇보다, 안 맞는 환경을 스스로 보정하며 점점 더 잘 치는 로봇을 보여주고자 한다.

---

## 1. 기존 당구 로봇과의 비교 관점

### 1.1 기존 당구 로봇들이 주로 해낸 것

| 기존 연구/사례의 강점 | 설명 |
|---|---|
| 공 위치 인식 | 카메라 기반으로 당구공 위치를 검출 |
| 포켓볼 shot selection | 어떤 공을 어떤 포켓에 넣을지 선택 |
| 전용 cue actuator 구현 | pneumatic cue, electromagnetic cue, gantry 구조 등 사용 |
| cue alignment | 큐 방향과 공 중심을 정렬 |
| stroke speed optimization | 타격 속도와 자세 최적화 |
| 사람 상대 전략 planning | 사람의 의사결정 모델을 고려한 게임 전략 |
| spin shot / cue ball positioning | 스핀을 이용해 cue ball의 최종 위치를 제어 |
| 캐롬 경로 assistive projection | 사람이 칠 수 있도록 3쿠션/캐롬 경로를 안내 |
| neural/fuzzy 보정 | 일부 연구에서 hitting error 보정 시도 |

### 1.2 기존 사례 대비 우리 팀이 노릴 수 있는 gap

기존 연구들이 전혀 하지 않았다고 주장하기보다는, **본 프로젝트는 다른 관점에 초점을 둔다**고 말하는 것이 안전하다.

| 구분 | 기존 당구 로봇 | 우리 팀 Indy7 당구 데모 |
|---|---|---|
| 주 목표 | 포켓볼 성공, shot selection, cue alignment | 트릭샷 경로 탐색 + sim-to-real 보정 |
| 하드웨어 | gantry, PR2, dual-arm, 전용 cue actuator | 범용 6축 로봇팔 Indy7 + 커스텀 툴팁 |
| 환경 | 표준 테이블 또는 잘 세팅된 실험 환경 | 저가형 미니 당구대, 불완전 마찰/쿠션 |
| 경로 계획 | 공-포켓 중심의 기하학적 shot planning | 로봇 실행 가능성 + 보정값 기반 path planning |
| 보정 | 일부 연구에서 vision/actuation 보정 | 반복 타격 결과 기반 self-calibration |
| 데모 포인트 | 공을 넣는 성능 | 실패-보정-재시도 과정과 오차 감소 |
| 차별성 | 잘 세팅된 환경에서 정확한 타격 | 안 맞는 환경을 로봇이 맞춰가는 과정 |

## 2. 우리 팀 차별점 5가지

### 2.1 Robot-aware shot planning

공의 경로만 계산하는 것이 아니라, Indy7이 실제로 그 샷을 수행할 수 있는지까지 판단한다.

고려할 요소:

- 로봇 작업공간
- inverse kinematics 가능 여부
- 관절 각도 제한
- ㄴ자형 툴팁의 접근 방향
- 당구대와의 충돌 가능성
- TCP 속도 제한
- 타격 방향과 tool tip 정렬 오차

발표용 문장:

> 이론적으로 가능한 샷이 아니라, Indy7이 실제로 칠 수 있고 오차에 강한 샷을 선택한다.

---

### 2.2 Calibration-based reflection model

이상적인 당구 모델은 입사각과 반사각이 같다고 가정하지만, 저가형 미니 당구대에서는 쿠션 반발 특성이 일정하지 않다.

따라서 각 쿠션별로 실제 반사각 오차를 측정하고, 경로 탐색 모델에 보정항을 넣을 수 있다.

예시:

| Cushion | 이상 반사각 | 실제 반사각 | 보정값 |
|---|---:|---:|---:|
| Left | 35° | 30° | -5° |
| Right | 35° | 38° | +3° |
| Top | 40° | 33° | -7° |
| Bottom | 40° | 36° | -4° |

발표용 문장:

> 저가형 미니 당구대에서는 쿠션별 반발 특성이 다르기 때문에, 본 알고리즘은 각 쿠션의 실제 반사 오차를 측정하고 이를 경로 탐색 단계에 보정항으로 반영한다.

---

### 2.3 Sim-to-real gap reduction

단순히 환경을 바꾸는 것이 아니라, 예측 경로와 실제 경로의 차이를 측정하고 줄이는 과정이 중요하다.

보여줄 수 있는 지표:

- 예측 경로 vs 실제 경로 overlay
- 목표 지점과 실제 도착점 사이 거리 오차
- 보정 전/후 평균 오차 비교
- 보정 전/후 성공률 비교
- 타격 속도-공 이동거리 보정 curve
- 쿠션 반사각 보정 table

발표용 문장:

> 단순히 잘 맞는 영상을 보여주는 것이 아니라, 시뮬레이션에서 예측한 공의 경로와 실제 공의 이동 경로를 비교하고, 반복 보정을 통해 그 차이가 줄어드는 과정을 보여준다.

---

### 2.4 Adaptive shot correction

로봇이 한 번 타격한 뒤 실제 결과를 측정하고, 다음 타격 조건을 업데이트한다.

기본 구조:

```text
Shot Planning
      ↓
Robot Strike
      ↓
Vision Measurement
      ↓
Prediction Error
      ↓
Parameter / Command Update
      ↓
Next Shot Planning
```

간단한 업데이트 식:

```text
v_next = v_current + k_d × distance_error
θ_next = θ_current + k_θ × angle_error
```

발표용 문장:

> 처음부터 복잡한 강화학습을 적용하기보다는, 반복 타격 데이터를 이용해 속도-거리 관계, 반사각 보정값, 타격 방향 보정값을 업데이트하는 adaptive calibration 방식으로 접근한다.

---

### 2.5 Trick shot demonstration

단순 직선 타격보다 쿠션샷, 장애물 통과, target zone 도달이 로보틱스 개념을 더 잘 보여준다.

보여줄 수 있는 개념:

- 공 위치 인식
- 좌표계 변환
- 경로 후보 생성
- 반사 모델
- 타격 방향 계산
- 로봇팔 IK
- TCP 속도 제어
- 결과 측정 및 보정

발표용 문장:

> 트릭샷은 단순히 멋있는 장면이 아니라, 로봇공학에서 배운 planning, kinematics, calibration, feedback 개념을 통합해서 보여줄 수 있는 task이다.

## 3. 경로 탐색 알고리즘 차별점

### 3.1 일반적인 당구 경로 탐색

일반적인 당구 경로 탐색은 다음 순서로 진행된다.

```text
현재 흰 공 위치
→ 목표 공 위치
→ 쿠션 반사 순서 선택
→ 반사점 계산
→ 타격 방향 계산
```

1쿠션의 경우, 목표점을 쿠션 너머로 대칭 이동한 뒤 흰 공에서 대칭 목표점까지 직선을 연결하면 반사점을 계산할 수 있다. 2쿠션, 3쿠션도 목표점을 여러 번 대칭 이동해 후보 경로를 생성할 수 있다.

하지만 이는 **수학적으로 가능한 경로**일 뿐, 로봇팔이 실제로 칠 수 있는지는 별개의 문제다.

---

### 3.2 우리 팀 알고리즘의 핵심

우리 팀 알고리즘의 차별점은 다음과 같다.

> 단순히 이론적인 당구 경로를 계산하는 것이 아니라, 로봇팔이 실제로 칠 수 있는 경로만 탐색하고, 실험 기반 보정값을 반영해 성공 가능성이 높은 경로를 선택한다.

추천 알고리즘 이름:

> Robot-Aware Calibrated Shot Planner

한국어:

> 로봇 실행 가능성을 고려한 보정 기반 샷 플래너

---

### 3.3 알고리즘 구조

```text
Ball Detection
      ↓
Coordinate Transformation
      ↓
Candidate Path Generation
      ↓
Reflection Model with Calibration
      ↓
Robot Feasibility Check
      ↓
Robustness Scoring
      ↓
Best Shot Selection
      ↓
Robot Trajectory Generation
      ↓
Execution
      ↓
Result Measurement
      ↓
Parameter / Score Update
```

---

### 3.4 후보 경로 scoring 기준

여러 경로 후보를 만든 뒤, 다음 기준으로 점수를 매긴다.

| 평가 기준 | 의미 |
|---|---|
| Target Accuracy | 예측상 목표에 가까운 정도 |
| Robot Feasibility | 로봇팔이 해당 자세로 칠 수 있는 정도 |
| Robustness | 작은 오차에도 덜 민감한 정도 |
| Speed Feasibility | 필요한 속도가 로봇 가능 범위 안에 있는 정도 |
| Empirical Success Rate | 과거 실험에서 성공한 비율 |

예시 scoring:

```text
Final Score =
0.35 × Target Accuracy
+ 0.25 × Robot Feasibility
+ 0.20 × Robustness
+ 0.10 × Speed Feasibility
+ 0.10 × Empirical Success Rate
```

발표용 문장:

> 본 알고리즘은 하나의 경로를 결정론적으로 선택하는 대신, 가능한 다중 쿠션 경로 후보를 생성하고 로봇 실행 가능성, 속도 조건, 쿠션 오차 민감도, 과거 성공률을 기준으로 가장 안정적인 후보를 선택한다.

---

### 3.5 1주일 안에 가능한 현실 버전

완전한 자동 경로 탐색이 어렵다면, 후보 경로를 3개 정도만 비교해도 충분하다.

예시:

| 후보 | 예측 오차 | 로봇 접근 가능 | 필요한 속도 | 최종 선택 |
|---|---:|---|---:|---|
| Path A: 왼쪽 쿠션 | 3 cm | 가능 | 0.4 m/s | 선택 |
| Path B: 위쪽 쿠션 | 5 cm | 불리 | 0.7 m/s | 제외 |
| Path C: 오른쪽 쿠션 | 4 cm | 가능 | 1.1 m/s | 제외 |

최소 구현 순서:

1. 1쿠션 또는 2쿠션 후보 경로 여러 개 생성
2. 각 경로의 예상 반사점 계산
3. 로봇이 칠 수 있는 방향인지 체크
4. 경로 길이와 필요한 속도 기준으로 후보 제거
5. 가장 짧고 안정적인 경로 선택
6. 실제 타격 후 오차 측정
7. 다음 시도에서 각도 또는 속도 보정

## 4. 교수님 질문 대비 답변 스크립트

### Q1. 기존 당구 로봇이랑 뭐가 다른데?

> 기존 당구 로봇들은 대부분 공 위치를 인식하고, 포켓에 넣기 위한 샷을 계산하고, 전용 cue mechanism이나 gantry 구조로 타격하는 방식이 많았습니다.  
> 저희 프로젝트의 차별점은 단순히 한 번 계산한 경로를 실행하는 것이 아니라, 저가형 미니 당구대에서 발생하는 마찰, 쿠션 반발, 툴팁 접촉 오차 때문에 생기는 sim-to-real gap을 직접 측정하고, 반복 타격 결과를 이용해 타격 조건을 보정한다는 점입니다.  
> 즉, 잘 세팅된 환경에서 정확히 치는 로봇보다는, 잘 안 맞는 환경을 스스로 맞춰가며 점점 더 잘 치는 로봇을 보여주는 것이 저희 팀의 목표입니다.

---

### Q2. 그냥 환경을 바꿔서 쉽게 만드는 거 아니야?

> 단순히 성공률을 높이기 위해 환경을 편하게 바꾸는 것이 아니라, 로봇이 다루는 물리 시스템의 불확실성을 줄이고 파라미터를 식별하기 위한 과정으로 보고 있습니다.  
> 현재 저가형 당구대는 표면 마찰이 너무 크거나 불균일해서, 같은 조건으로 쳐도 공이 직선으로 가지 않거나 중간에 멈추는 문제가 있습니다. 이 상태에서는 경로 탐색 알고리즘의 성능을 평가하기 어렵습니다.  
> 그래서 바닥과 쿠션을 바꾸는 것은 이상적인 당구대처럼 만들기 위한 것이 아니라, 반복 가능한 실험 조건을 만들고 그 위에서 마찰, 반사각, 공 이동거리 같은 파라미터를 측정하기 위한 전처리라고 생각합니다.

추가 방어 문장:

> 제어 실험에서도 센서 노이즈나 기구적 백래시가 너무 크면 먼저 시스템을 안정화한 뒤 모델링을 합니다. 저희도 같은 관점에서 당구대를 로봇 시스템의 일부로 보고, 환경을 안정화한 후 sim-to-real gap을 정량적으로 줄이는 방향으로 접근하고 있습니다.

---

### Q3. 경로 탐색 알고리즘에서 차별점이 뭐야?

> 일반적인 당구 경로 탐색은 공과 쿠션의 기하학적 관계만 보고 반사점을 계산합니다. 하지만 로봇팔이 실제로 그 경로를 칠 수 있는지는 별개의 문제입니다.  
> 저희 알고리즘은 후보 경로를 여러 개 생성한 뒤, 단순히 이론적으로 가능한 경로가 아니라 Indy7이 실제로 실행 가능한 경로를 선택합니다.  
> 예를 들어 타격 방향이 계산되더라도 로봇팔의 작업공간 밖에 있거나, ㄴ자형 툴팁이 해당 방향으로 접근하기 어렵거나, 필요한 TCP 속도가 너무 높으면 그 경로는 제외합니다.  
> 따라서 저희의 차별점은 공의 경로 탐색과 로봇팔 실행 가능성 판단을 결합한 shot planning입니다.

---

### Q4. sim-to-real gap을 줄였다는 걸 어떻게 보여줄 거야?

> sim-to-real gap을 줄였다는 것을 단순히 말로 설명하지 않고, 보정 전후 데이터를 비교해서 보여주려고 합니다.  
> 첫 번째로, 시뮬레이션에서 예측한 공의 경로와 실제 카메라로 측정한 공의 이동 경로를 하나의 이미지 위에 overlay해서 비교할 수 있습니다.  
> 두 번째로, 목표 지점과 실제 공의 도착 지점 사이의 거리 오차를 측정하고, 보정 전후 평균 오차가 얼마나 줄었는지 보여줄 수 있습니다.  
> 세 번째로, 1쿠션이나 2쿠션 target zone 도달 실험에서 보정 전 성공률과 보정 후 성공률을 비교할 수 있습니다.  
> 즉, 저희는 성공 장면 하나보다 예측 오차가 줄어드는 과정을 결과로 보여주려 합니다.

---

### Q5. 로봇이 스스로 보정한다는 게 정확히 무슨 뜻이야?

> 여기서 스스로 보정한다는 것은 로봇이 한 번 타격한 뒤 실제 공의 결과를 측정하고, 시뮬레이션 예측과 실제 결과의 차이를 계산한 다음, 다음 타격 조건에 반영하는 것을 의미합니다.  
> 예를 들어 목표 지점보다 공이 짧게 갔다면 다음 시도에서는 TCP 속도를 증가시키고, 목표보다 오른쪽으로 빗나갔다면 다음 시도에서는 타격각을 반대 방향으로 보정합니다.  
> 처음부터 강화학습처럼 복잡한 policy를 학습하는 것이 아니라, 반복 타격 데이터를 이용해 속도-거리 관계, 반사각 보정값, 타격 방향 보정값을 업데이트하는 adaptive calibration 방식입니다.

---

### Q6. 학습이라고 말할 수 있어?

> 엄밀한 의미의 강화학습이라고 주장하기보다는, 반복 실험 데이터를 이용한 learning-based correction 또는 adaptive calibration이라고 표현하는 것이 더 정확하다고 생각합니다.  
> 로봇이 매 trial마다 입력 명령, 실제 공 이동 결과, 목표 오차를 저장하고, 그 데이터를 이용해 다음 타격 속도나 각도를 수정합니다.  
> 즉, 상태-action-reward를 갖춘 RL까지는 아니더라도, 실험 데이터를 통해 모델 파라미터와 명령 보정값이 업데이트되고 결과 오차가 줄어든다면 넓은 의미에서 학습 기반 보정이라고 볼 수 있습니다.

추가 방어 문장:

> 지금 단계에서는 데이터 수가 많지 않기 때문에 딥러닝보다는 lookup table, 선형 보정, gain-based update처럼 해석 가능한 방식이 더 적합하다고 판단했습니다.

---

### Q7. 그냥 trial-and-error 아닌가?

> 단순 trial-and-error와의 차이는, 저희는 실패 결과를 그냥 보고 감으로 수정하는 것이 아니라, 목표 위치와 실제 위치의 오차를 계산하고 그 오차를 다음 명령에 정량적으로 반영한다는 점입니다.  
> 예를 들어 실제 공이 목표보다 8 cm 짧게 갔다면 속도 보정항을 추가하고, 반사 후 오른쪽으로 5 cm 벗어났다면 타격각 보정항을 적용합니다.  
> 즉, 시행착오를 하더라도 그 과정이 데이터로 기록되고, 다음 경로 탐색과 타격 조건에 반영되는 closed-loop 보정 구조입니다.

---

### Q8. 왜 Indy7이어야 해?

> 전용 gantry나 cue actuator를 사용하면 당구 task에 맞게 설계되어 있기 때문에 타격 자체는 더 쉬울 수 있습니다.  
> 하지만 Indy7은 원래 당구 전용 로봇이 아니기 때문에, 타격점을 만들기 위해 좌표계 변환, 역기구학, 작업공간 판단, 엔드이펙터 설계가 모두 필요합니다.  
> 그래서 저희 프로젝트는 단순한 당구 자동화가 아니라, 범용 로봇팔을 이용해 동적 접촉 task를 수행하기 위한 통합 로봇 시스템 구현이라는 점에서 의미가 있습니다.

---

### Q9. 성공률이 낮으면 어떡할 거야?

> 3쿠션 같은 고난도 샷은 성공률이 낮을 수 있습니다. 그래서 저희는 최종 성능을 단순히 성공/실패 하나로 평가하기보다는, 보정 전후 오차가 줄어드는지를 핵심 지표로 두려고 합니다.  
> 예를 들어 처음에는 목표 지점에서 크게 벗어나더라도, 반복 보정 후 목표와의 거리가 줄어든다면 로봇이 환경에 적응하고 있다는 것을 보여줄 수 있습니다.  
> 따라서 최종 데모는 성공 장면뿐만 아니라, 실패 결과를 측정하고 다음 시도에서 개선되는 과정을 함께 보여주는 방향으로 구성하려고 합니다.

---

## 5. 현실적으로 1주일 안에 추가해서 보여줄 수 있는 요소들

남은 기간이 1주일 조금 넘는다면, 모든 것을 완성하려고 하기보다 **보여줄 수 있는 증거**를 만드는 것이 중요하다.

추천 우선순위:

1. 경로 탐색이 보이는 최소 알고리즘
2. sim-to-real gap을 줄이는 보정 과정
3. 성공 가능성이 높은 트릭샷
4. 보정 전/후 비교 자료

---

## 5.1 경로 탐색: 1주일 안에 가능한 버전

### 목표

완전 자동 경로 탐색이 아니라, **후보 경로를 여러 개 만들고 그중 하나를 선택하는 과정**을 보여준다.

### 구현 가능 데모

#### A. 1쿠션 후보 3개 비교

공과 목표 지점을 정해두고, 다음 후보를 만든다.

| 후보 | 설명 |
|---|---|
| Path A | 왼쪽 쿠션 반사 |
| Path B | 위쪽 쿠션 반사 |
| Path C | 오른쪽 쿠션 반사 |

각 후보에 대해 다음 항목을 계산 또는 수동 평가한다.

| 평가 항목 | 방법 |
|---|---|
| 경로 길이 | 짧을수록 마찰 오차가 작음 |
| 로봇 접근 가능성 | Indy7이 해당 타격 방향을 만들 수 있는지 확인 |
| 필요한 속도 | 너무 세거나 약하면 제외 |
| 목표 지점과의 예측 오차 | 반사 모델 기준으로 목표와 얼마나 가까운지 |
| 쿠션 오차 민감도 | 반사점이 너무 예민한 위치면 제외 |

발표용 결과 예시:

| 후보 | 경로 길이 | 로봇 접근 | 필요 속도 | 최종 판단 |
|---|---:|---|---:|---|
| Left cushion | 짧음 | 가능 | 0.4 m/s | 선택 |
| Top cushion | 중간 | 가능 | 0.7 m/s | 보류 |
| Right cushion | 김 | 불리 | 1.0 m/s 이상 | 제외 |

### 발표 포인트

> 단일 경로를 찍어서 치는 것이 아니라, 가능한 경로 후보를 만들고 로봇 실행 가능성과 안정성을 기준으로 선택했다.

---

### B. Robot feasibility check만 추가해도 차별점이 됨

완전 자동 IK 체크가 어렵다면, 다음만 해도 충분하다.

- 후보 타격 방향별로 로봇이 접근 가능한지 체크
- ㄴ자 툴팁이 공 중심선에 정렬 가능한지 체크
- 당구대와 충돌 위험이 큰 후보 제외
- TCP 속도가 지나치게 큰 후보 제외

이 내용을 경로 선택표에 넣으면 **로봇팔 기반 경로 탐색**이라는 차별점이 생긴다.

---

## 5.2 Sim-to-real gap / 학습 과정: 1주일 안에 가능한 버전

### 목표

복잡한 강화학습이 아니라, **실험 데이터를 이용해 모델 또는 명령값이 업데이트되는 과정**을 보여준다.

가장 현실적인 표현:

> Learning-based correction 또는 adaptive calibration

---

### A. 속도-거리 lookup table 만들기

가장 쉽고 효과적인 실험이다.

#### 방법

1. 같은 위치에서 흰 공을 둔다.
2. TCP 속도를 3~5개로 나눠서 타격한다.
3. 각 속도마다 공 이동거리를 측정한다.
4. 평균과 표준편차를 계산한다.
5. 목표 거리별 필요한 TCP 속도를 lookup table로 만든다.

#### 추천 속도

| Trial set | TCP 속도 |
|---|---:|
| Set 1 | 0.2 m/s |
| Set 2 | 0.3 m/s |
| Set 3 | 0.4 m/s |
| Set 4 | 0.5 m/s |

#### 결과 표 예시

| TCP 속도 | 평균 이동거리 | 표준편차 |
|---:|---:|---:|
| 0.2 m/s | 18 cm | 2.1 cm |
| 0.3 m/s | 31 cm | 2.8 cm |
| 0.4 m/s | 47 cm | 3.5 cm |
| 0.5 m/s | 66 cm | 4.2 cm |

#### 발표 포인트

> 로봇 명령 속도와 실제 공 이동거리 사이의 관계를 실험적으로 측정하여, 시뮬레이션의 타격 세기 모델을 실제 환경에 맞게 보정했다.

---

### B. 1쿠션 반사각 보정

#### 방법

1. 공을 쿠션 방향으로 친다.
2. 카메라 영상 또는 위에서 찍은 사진에서 입사 경로와 반사 경로를 선으로 표시한다.
3. 이상 반사각과 실제 반사각 차이를 계산한다.
4. 쿠션별 보정값을 만든다.

#### 결과 표 예시

| Cushion | Ideal reflection | Real reflection | Correction |
|---|---:|---:|---:|
| Left | 35° | 30° | -5° |
| Right | 35° | 38° | +3° |
| Top | 40° | 33° | -7° |
| Bottom | 40° | 36° | -4° |

#### 발표 포인트

> 이상적인 입사각=반사각 모델이 실제 미니 당구대에서는 맞지 않았고, 각 쿠션별 반사 오차를 측정하여 경로 탐색 모델에 보정항으로 반영했다.

---

### C. 보정 전/후 목표 오차 비교

#### 방법

1. 1쿠션 target zone을 설정한다.
2. 보정 전 모델로 5~10회 타격한다.
3. 목표 지점과 실제 도착점의 거리 오차를 기록한다.
4. 반사각 또는 타격각 보정을 적용한다.
5. 보정 후 같은 조건으로 5~10회 타격한다.
6. 평균 오차와 성공률을 비교한다.

#### 결과 표 예시

| Method | 평균 목표 오차 | 성공 횟수 / 10회 | 성공률 |
|---|---:|---:|---:|
| Ideal model only | 14.2 cm | 3/10 | 30% |
| Calibrated model | 5.5 cm | 7/10 | 70% |

#### 발표 포인트

> 성공 장면 하나가 아니라, 보정 전후 평균 오차와 성공률 변화를 통해 sim-to-real gap이 줄어드는 것을 정량적으로 보여준다.

---

### D. 반복 trial에서 오차 감소 curve

자가 보정 느낌을 가장 잘 보여주는 자료다.

#### 방법

1. 같은 1쿠션 target zone 샷을 반복한다.
2. 매 trial마다 실제 도착점 오차를 측정한다.
3. 다음 trial에서 각도 또는 속도를 보정한다.
4. trial이 진행될수록 오차가 감소하는지 확인한다.

#### 결과 예시

| Trial | Target error |
|---:|---:|
| 1 | 15.0 cm |
| 2 | 9.2 cm |
| 3 | 5.8 cm |
| 4 | 3.1 cm |

#### 발표 포인트

> 로봇은 각 타격 후 실제 도착 위치를 측정하고, 목표와의 오차를 바탕으로 다음 타격 조건을 갱신하였다. 그 결과 반복 시도에 따라 목표 오차가 감소하였다.

---

## 5.3 트릭샷: 1주일 안에 가능한 후보

### 우선순위 1. 장애물 사이 통과 후 목적구 맞히기

#### 구성

```text
cue ball → 장애물 두 개 사이 통과 → target ball
```

#### 장점

- 성공 가능성이 높음
- 관객이 보기 좋음
- path planning / collision avoidance 느낌이 강함
- 쿠션 반사보다 물리 불확실성이 작음

#### 성공 기준

처음에는 gap을 넓게 설정한다.

```text
공 지름의 3~4배 정도 gap
```

성공하면 gap을 줄여 난이도를 올린다.

#### 발표 포인트

> 로봇은 목표 공을 향한 직선 경로만 선택하는 것이 아니라, 장애물과의 충돌을 고려하여 실행 가능한 타격 경로를 선택한다.

---

### 우선순위 2. 1쿠션 target zone 도달

#### 구성

```text
cue ball → cushion 1번 반사 → target zone 도달
```

#### 장점

- sim-to-real gap 보정 설명이 좋음
- 쿠션 반사 모델을 보여줄 수 있음
- 2쿠션/3쿠션으로 확장 가능
- 성공 기준을 target zone으로 잡으면 성공률 확보 가능

#### 성공 기준

```text
target zone 반지름 5~8 cm 안에 들어오면 성공
```

#### 발표 포인트

> 이상적인 반사 모델로는 target zone에서 벗어났지만, 실제 반사각 보정 후 목표 지점에 더 가까워졌다.

---

### 우선순위 3. 2쿠션 target zone 도달

#### 구성

```text
cue ball → cushion 1 → cushion 2 → target zone
```

#### 장점

- multi-cushion path planning이라고 말할 수 있음
- 3쿠션보다 현실적
- 최종 challenge demo로 적합

#### 성공 기준

목적구 맞히기보다는 target zone 도달로 잡는다.

```text
target zone 반지름 8~12 cm
```

#### 발표 포인트

> 3쿠션은 최종 목표로 두고, 현재 데모에서는 이를 위한 핵심 단계인 2쿠션 경로 탐색과 보정 가능성을 검증한다.

---

### 우선순위 4. 3쿠션 challenge demo

#### 구성

```text
cue ball → cushion 1 → cushion 2 → cushion 3 → target zone 또는 target ball
```

#### 장점

- 가장 임팩트 있음
- 최종 목표로 사용 가능

#### 단점

- 성공 가능성 낮음
- 쿠션/마찰 오차 누적이 큼
- 남은 기간상 메인 데모로 걸기에는 위험함

#### 추천 표현

> 3쿠션은 최종 목표로 제시하고, 최종 발표에서는 1쿠션/2쿠션 보정 결과와 함께 challenge demo로 시도한다.

## 6. 툴팁 개선 방향

현재 툴팁이 PLA 3D printing 구조라면, PLA가 공에 직접 닿는 것은 추천하지 않는다. PLA는 단단하고 미끄러우며, 실제 당구 큐팁처럼 안정적인 마찰을 만들기 어렵다.

추천 구조:

```text
[PLA body]
   ↓
[평평한 부착면]
   ↓
[얇은 고무 완충층 0.5~1 mm]
   ↓
[당구 큐팁 가죽 또는 단단한 고무]
   ↓
[공]
```

추천 재료:

| 재료 | 추천도 | 이유 |
|---|---:|---|
| PLA 단독 | ★ | 미끄럽고 접촉 불안정 |
| PLA + 고무패드 | ★★★★ | 구현 쉽고 안정적 |
| PLA + 큐팁 가죽 | ★★★★★ | 실제 큐와 가장 유사 |
| PLA + 큐팁 가죽 + 초크 | ★★★★ | 마찰 좋지만 스핀 주의 |
| PLA + 실제 큐대 끝부분 | ★★★★★ | 가장 그럴듯하지만 제작 난이도 있음 |
| PLA + 스펀지/EVA | ★★ | 힘 전달 불안정 가능 |

발표용 문장:

> PLA 구조물은 엔드이펙터의 형상과 강성을 확보하기 위한 프레임으로 사용하고, 실제 공과 접촉하는 부분에는 당구 큐팁과 유사한 가죽 또는 고무 재질을 부착하여 접촉 마찰과 충격 전달 안정성을 높였다.

툴팁 개선도 sim-to-real gap reduction의 일부로 설명할 수 있다.

> 로봇-공 접촉 모델의 불확실성을 줄이기 위해 접촉 재료를 개선했다.

## 7. 최종 발표용 핵심 문장 모음

### 프로젝트 한 문장

> 본 프로젝트의 핵심은 로봇팔이 공을 한 번 치는 장면이 아니라, 공의 경로를 예측하고, 실제 결과를 측정하고, 그 오차를 줄여가며 더 나은 샷을 선택하는 과정이다.

### 차별점 한 문장

> 우리는 이론적으로 가능한 경로가 아니라, Indy7이 실제로 칠 수 있고 오차에 강한 경로를 선택한다.

### sim-to-real 한 문장

> 단순히 환경을 바꾸는 데 그치지 않고, 타격 속도와 실제 이동거리, 이상 반사각과 실제 반사각, 목표 지점과 실제 도착점 오차를 측정하여 모델을 보정하였다.

### 학습/보정 한 문장

> 복잡한 강화학습보다는 반복 타격 데이터를 이용한 adaptive calibration으로 접근하여, 매 trial마다 오차를 측정하고 다음 타격 조건을 업데이트한다.

### 최종 목표 한 문장

> 저희는 범용 로봇팔이 불완전한 물리 환경에서도 경로 탐색과 보정을 통해 동적 접촉 task를 수행할 수 있음을 보여주고자 한다.

## 8. 발표 슬라이드 추천 구성

1. **Motivation: Why robotic billiards?**
   - 동적 접촉, 마찰, 반사, 타격 제어가 포함된 로봇 task

2. **Related Work & Gap**
   - 기존 당구 로봇 vs 우리 팀 접근

3. **System Overview**
   - Camera → path planning → robot trajectory → strike → measurement → correction

4. **Robot-Aware Calibrated Shot Planner**
   - 후보 경로 생성, 로봇 실행 가능성, 보정값 기반 scoring

5. **Sim-to-Real Calibration**
   - 속도-거리 calibration, 쿠션 반사각 보정, 도착점 오차 비교

6. **Custom End-effector**
   - ㄴ자형 tool tip, PLA body, 큐팁/고무 접촉부

7. **Demo Results**
   - 직선 타격, 장애물 통과, 1쿠션 target zone, 보정 전/후

8. **Limitations & Future Work**
   - 2쿠션/3쿠션 확장, 더 많은 데이터 기반 학습, 실시간 재계획

## 9. 최종 결론

1주일 안에 가장 좋은 전략은 “3쿠션 무조건 성공”을 메인으로 걸기보다, **경로 탐색과 sim-to-real 보정이 실제로 보이는 데모**를 만드는 것이다.

최종적으로는 다음 세 가지를 확보하는 것이 가장 좋다.

1. **경로 탐색**  
   - 후보 경로 2~3개 비교 후 robot-aware scoring으로 선택

2. **sim-to-real 보정**  
   - 속도-거리 lookup table, 쿠션 반사각 보정, 보정 전/후 오차 비교

3. **트릭샷**  
   - 장애물 사이 통과 후 목적구 맞히기  
   - 1쿠션 target zone 도달  
   - 가능하면 2쿠션 target zone 또는 3쿠션 challenge

최종 메시지:

> 본 프로젝트는 단순히 로봇팔로 당구공을 타격하는 데모가 아니라, 불완전한 미니 당구대 환경에서 로봇이 경로를 탐색하고, 실제 타격 결과를 바탕으로 모델과 타격 조건을 보정하며, 점진적으로 트릭샷 성공률을 높이는 자가 보정형 로봇 당구 시스템이다.

## 🎯 프로젝트 개요

### 목표
Indy7 로봇 팔에 ㄴ자형 타격 도구를 장착하고, **자율 상태 머신(SCAN → THINK → STRIKE → OBSERVE)**을 통해 3쿠션 당구를 수행합니다.

- 수구(흰공)로 황구·적구 모두 접촉
- 총 쿠션 반사 2회 이상 (2쿠션/3쿠션 모두 유효)
- 장애물 없이 3회 시도 → **2/3 성공률** 달성

### 핵심 기술
| 기술 | 설명 |
|------|------|
| **PyBullet 공 전용 시뮬레이션** | 720각도 × 3속도 = 2160개 후보를 헤드리스 PyBullet에서 직접 탐색 |
| **각도 우선순위 정렬** | 실측 기반 안전 각도(50-180°) 우선 선택으로 글랜싱 블로우 제거 |
| **적은 쿠션 우선** | 쿠션 수가 적은 후보를 우선 선택하여 궤적 예측 정확도 향상 |
| **Damped Least Squares IK** | 5회 반복 수렴 + 관절한계/특이점/바디뚫림 검증 |
| **PD 토크 제어** | Kp=5000, Kd=200 부스트로 1.0 m/s 수평 타격 달성 |
| **ㄴ자 도구 수평 타격** | TOOL_VERTICAL_DROP=6cm + TOOL_HORIZONTAL_EXT=3cm 오프셋 자동 계산 |
| **즉시 후퇴 프로토콜** | 타격 직후 Home 복귀로 공 궤적 방해 방지 |

## 🏗️ 시스템 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                  State Machine (FSM)                    │
│  ┌──────┐   ┌───────┐   ┌────────┐   ┌─────────┐      │
│  │ SCAN │──▶│ THINK │──▶│ STRIKE │──▶│ OBSERVE │──┐   │
│  └──────┘   └───────┘   └────────┘   └─────────┘  │   │
│      ▲                                      │      │   │
│      └──────── (retry if miss) ─────────────┘      │   │
│      └──────── (next round if success) ────────────┘   │
└────────────┬───────────────┬──────────────┬────────────┘
             │               │              │
     ┌───────▼───────┐ ┌────▼────┐  ┌──────▼──────┐
     │ CushionPlanner│ │Traj.Plan│  │ RobotControl│
     │ (공 시뮬)     │ │(SE3 궤적)│  │ (PD 토크)   │
     └───────────────┘ └─────────┘  └─────────────┘
             │                              │
     ┌───────▼───────┐              ┌──────▼──────┐
     │ PyBullet      │              │ Pinocchio   │
     │ (Headless)    │              │ (FK, Jac)   │
     └───────────────┘              └─────────────┘
```

### 작동 흐름

1. **SCAN**: PyBullet에서 3공 위치 + 테이블 bounds 읽기
2. **THINK**: 720각도 × 3속도 공 전용 시뮬레이션 → 안전 각도 + 적은 쿠션 우선 정렬 → 최대 25개 후보
3. **STRIKE**: 후보별 IK 검증 → 첫 유효 후보 실행 → 즉시 Home 복귀
4. **OBSERVE**: 접촉 이벤트 추적 → 총 쿠션 ≥ 2 + 양 타겟 히트 → 성공

## 🔍 핵심 모듈

### 1. `cushion_planner.py` — 공 전용 시뮬 기반 탐색

```
1. Grid Search: 720각도(0.5°) × 3속도 = 2160개 후보
2. PyBullet 헤드리스에서 3공 직접 시뮬레이션
3. 각도 우선순위 정렬 (글랜싱 블로우 방지):
   - 50-180°: 안전 (우선순위 0) ← 항상 정확한 타격
   - 30-50°, 180-330°: 보통 (우선순위 1)
   - 0-30°, 330-360°: 위험 (우선순위 2) ← 글랜싱 블로우 빈발
4. 같은 우선순위 내에서 쿠션 수 적은 것 우선 (궤적 예측 정확도 ↑)
5. 5° 각도 다양성 필터 → 최대 25개 후보
```

### 2. `state_machine.py` — 자율 상태 머신

**성공 기준**: 양 타겟 히트 + 전체 이벤트에서 총 쿠션 ≥ 2 (2쿠션/3쿠션 모두 유효)

**후보 선택**: IK 검증 통과하는 첫 후보 실행
- Manipulability ≥ 0.015 (글랜싱 블로우 방지)
- 바디 뚫림 z < 0.30m
- 관절 점프 < 0.3 rad

### 3. `trajectory_planner.py` — ㄴ자 도구 수평 타격

```
ㄴ자 도구 형상:
  EE
   |  (6cm 수직)
   |
   └──● (3cm 수평, 큐팁)

궤적: Home → 상공(25cm) → Ready(공 뒤 8cm) → Impact(공) → Follow(5cm) → Home
```

### 4. `ik_solver.py` — 역기구학 솔버

- 5회 반복 DLS IK 수렴 (1회만 하면 곡선 궤적 발생)
- 궤적 전체에서 10% 샘플링하여 검증
- 관절한계, 특이점, 바디뚫림, 관절점프 4단계 검사

## 🚀 설치 및 실행

### 요구 사항
```
Python 3.10+, PyBullet, NumPy, Matplotlib, Pinocchio
```

```bash
conda activate indy7_project
cd code/

# 3쿠션 당구 (장애물 없음, 3회 시도)
python run_maze.py --obstacles 0 --attempts 3

python run_maze.py --obstacles 5 --attempts 3
```

## 📊 성능 및 최적화

### 달성 성능
| 지표 | 값 |
|------|-----|
| **성공률** | 2/3 (66%) — 장애물 없음, 3회 시도 |
| **타격 각도 정확도** | 0.0-0.3° (fast search 대비) |
| **공 속도** | 1.836-1.862 m/s (일관적) |
| **글랜싱 블로우** | 0% (각도 우선순위 정렬 후) |
| **탐색 시간** | ~3초 (2160개 후보) |

### 최적화 히스토리
| 문제 | 해결 |
|------|------|
| 글랜싱 블로우 (60° 각도 오차) | 각도 0-30° 위험 구간 발견 → 50-180° 우선 정렬 |
| Fast search ↔ GUI 궤적 불일치 | 적은 쿠션 우선 선택 (반사 적을수록 예측 정확) |
| 성공 기준 불일치 | fast search와 state_machine 모두 전체 쿠션 수 기준 통일 |
| IK 곡선 궤적 | IK 5회 반복 수렴으로 직선 보장 |
| PD 속도 부족 | Kp=5000, Kd=200 부스트 |

### 남은 한계
1. **궤적 불일치**: 3+ 쿠션 반사 후 fast search와 GUI의 궤적이 분기 (공 스핀 미반영)
2. **극단적 공 위치**: 테이블 코너 근처에서 도달 가능한 후보 부족
3. **비전 미구현**: PyBullet API로 위치 직접 읽음

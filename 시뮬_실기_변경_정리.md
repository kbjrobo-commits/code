# 시뮬레이션 · 실기 변경 사항 정리

이 문서는 최근 대화에서 반영한 코드 변경과, **PyBullet 시뮬** vs **Indy 실제 로봇·카메라** 에 각각 어떻게 적용되는지 정리합니다.

---

## 1. 변경 요약 (한눈에)

| 영역 | 변경 파일 | 시뮬 | 실기 |
|------|-----------|------|------|
| 사이드 포켓(홀) 회피 | `config.py`, `pocket_avoidance.py`, `cushion_planner.py` | 탐색 필터만 (구멍 없음) | 탐색 필터만 (물리 홀 그대로) |
| 비전 기반 공 배치 | `calibration_loop_fix.py` | PyBullet 공 위치 = 카메라 | 카메라 검출 |
| 실기↔시뮬 로봇 동기화 | `calibration_loop_fix.py` | GUI 팔 = 실기 관절각 | Indy `get_control_data()['q']` |
| 실기 타격 재생 (movel) | `calibration_loop_fix.py` | `pb.MoveRobot` (관절) | `indy.movel` (TCP 직선) |
| movel 대기 로직 | `calibration_loop_fix.py` | — | 모션 시작·완료 대기 |
| 물리 파라미터 (권장) | `config.py` / 캘리브 시드 | PyBullet 마찰·반발 | 캘리브 후 시뮬 보정 |

## 2. 사이드 포켓(홀) 회피 — 경로 탐색

### 2.1 무엇이 바뀌었나

**가정:** 실물 당구대 **긴 변(y±, 폭 0.63 m) 레일 정중앙**에 사이드 포켓 2곳. 레일 방향(X) 개구부 **약 5 cm**. 짧은 변(x±) 모서리 홀은 없음.

**추가/수정 파일:**

| 파일 | 역할 |
|------|------|
| `project/config.py` | `MAZE_SIDE_POCKET_*` 상수 (회피 ON/OFF, 홀 길이, 안쪽 깊이, 마진) |
| `project/physics/pocket_avoidance.py` | **신규** — 1쿠션 기하 예측, 금지 구역, 각도 reject |
| `project/physics/cushion_planner.py` | Grid/refine 탐색·후보 필터에 `_reject_side_pocket_path()` 적용 |

**알고리즘 (탐색 전, PyBullet 없이):**

1. 큐볼 위치 + 타격 각도 → **첫 쿠션** 충돌점·반사 방향 계산 (`ray_first_cushion`)
2. 아래면 해당 각도 **후보에서 제외** (`rejects_first_cushion_toward_side_pocket`):
   - 첫 쿠션이 **긴 변(y±) 중앙 홀 구간**에 닿음
   - 첫 쿠션 **반사 후** 궤적이 홀 회피 박스(AABB)를 지남

**현재 테이블 기준 홀 중심 (월드 m):**

- y+ 긴 변: **(0.485, 0.477)**
- y− 긴 변: **(0.485, −0.153)**

X = `(x_min + x_max) / 2` (테이블 bounds 중앙).

### 2.2 시뮬레이션에서

| 항목 | 반영 |
|------|------|
| PyBullet 테이블/쿠션 mesh | **변경 없음** — 홀 geometry 없음 |
| `CushionShotPlanner._fast_ball_search` | reject된 각도는 **시뮬 돌리지 않음** |
| Headless fast sim (공 직접 속도) | 통과한 각도만 `score` 계산 |
| GUI `MazeEnvironment` | 홀 없음; 공이 홀로 빠지는 물리 **없음** |

→ 시뮬에서는 **“홀 근처로 가는 샷을 고르지 않게”** 만든 것이지, 구멍에 빠지는 물리를 추가한 것이 **아님**.

### 2.3 현실에서

| 항목 | 반영 |
|------|------|
| 실물 홀 | 그대로 존재 (누르면 움푹 들어가는 쿠션 등) |
| 로봇/Indy | 홀 회피 API **없음** — 플래너가 고른 각도만 실행 |
| 카메라 | 홀 회피와 **무관** (공 위치·최종 위치만) |

→ 실기에서도 **경로 선택**만 바뀌고, 홀에 빠질 위험은 줄어들지만 **완전 차단은 아님** (오차·쿠션 변형 시).

### 2.4 설정 (`project/config.py`)

```python
MAZE_SIDE_POCKET_AVOID = True          # False → 회피 끔
MAZE_SIDE_POCKET_HALF_LENGTH = 0.025   # 레일 방향 반길이 → 개구부 ~5 cm
MAZE_SIDE_POCKET_INWARD_DEPTH = 0.07   # 레일에서 안쪽 회피 깊이
MAZE_SIDE_POCKET_MARGIN = 0.012        # 추가 여유
```

실행 시 로그 예:

```text
[PocketAvoid] 긴 변(y±) 중앙 홀 2곳 (가정): y+ (0.485, 0.477), y- (0.485, -0.153)
[PocketAvoid] grid rejected N (1st-cushion → side pocket zone)
```

## 3. `calibration_loop_fix.py` — 캘리브레이션 · 실기 재생

### 3.1 비전 전용 공 배치

**변경:** `env.setup(num_obstacles=0)` 고정 좌표 대신 `_setup_env_from_vision()` / `_detect_balls_vision()`.

| | 시뮬 | 실기 |
|---|------|------|
| 공 시작 위치 | 카메라 → `env.reset_balls()` | 동일 (실제 공) |
| z 높이 | `MAZE_TABLE_SURFACE_HEIGHT` 등 config 식 | 비전은 x,y만; z는 config |

### 3.2 실기 ↔ GUI 시뮬 로봇 동기화

**추가 함수:** `_sync_sim_from_real`, `_movej_both`

| | 시뮬 | 실기 |
|---|------|------|
| `_sync_sim_from_real` | Indy 관절각 → `resetJointState` | `get_control_data()['q']` 읽기 |
| `_movej_both` | `pb` 관절 갱신 | `indy.movej` |
| `_wait_indy(..., sync_sim=True)` | 대기 중 GUI 팔 추적 | `is_in_motion` 대기 |

→ 시뮬 GUI가 **실기 팔 위치**와 맞춰 보이도록 함 (타격 물리 자체를 같게 만드는 것은 아님).

### 3.3 `_wait_indy` 개선

**변경:** `6TestCode_RealRobot.wait_indy` 와 동일 패턴.

1. `movel`/`movej` 후 **0.2 s** 대기  
2. 최대 **3 s** 동안 `is_in_motion == True` 될 때까지 대기  
3. **완료**까지 대기  

| | 시뮬 | 실기 |
|---|------|------|
| 적용 | `sync_sim` 시에만 부수적 | **모든** movej/movel 후 |

→ 실기에서 “명령만 보내고 다음 단계로 넘어가 접근만 하고 홈” 현상 완화.

### 3.4 `_replay_strike_on_real` (실기 타격)

**변경 전 (일반 paste 버전):** movej 접근 + hybrid movel + 함수 끝 `movej(HOME)`.

**변경 후 (`calibration_loop_fix.py`):**

```
Approach  — SE3 waypoint → indy.movel (느림)
Align     — ready → movel
Strike    — ready→follow 직선 → movel (플래너 SE3 delta)
Retract   — z만 상승 → movel
(홈은 호출부 _return_home 에서만)
```

| 단계 | 시뮬 (`calibration_loop_fix`) | 실기 |
|------|-------------------------------|------|
| Approach | `pb.MoveRobot(q)` 연속 | `indy.movel` (SE3 mm) |
| Strike | `pb.MoveRobot` strike+follow 구간 | **단일** `indy.movel` 직선 |
| Retract | (시뮬은 follow까지) | `RETRACT_HEIGHT` 만큼 z↑ movel |
| Home | — | `_return_home` → `movej(HOME_Q_DEG)` |

**Strike skip 방지:** TCP가 ready 근처인데 플래너 Δ ≥ 3 mm 이면 **현재 TCP 기준**으로 strike 목표 재계산.

**진단 로그:** `STRIKE DIAG`, `Strike movel`, `Strike 이동량`.

| | 시뮬 | 실기 |
|---|------|------|
| API | PyBullet 관절 제어 | IndyDCP3 `movel` / `movej` |
| 성공 의미 | GUI에서 팔·공 움직임 | TCP가 목표 근처 도달 |
| 불일치 원인 | IK/PD 이상적 | 좌표·모드·3 mm skip·movel 거부 |

### 3.5 Phase 1 / Phase 2 흐름 (fix 파일)

| Phase | 시뮬 | 실기 |
|-------|------|------|
| **1 좌표** | 시뮬 strike 재생 + `_replay_strike_on_real` | 동일 + 키보드 `[l/r/g/m]` |
| **2 물리** | `CushionShotPlanner` (홀 회피 포함) → 시뮬 → 실기 | 동일 → 카메라 최종 위치 → Nelder-Mead |

## 4. 물리 파라미터 (색지 1T · 말랑 쿠션 권장 시드)

`config.py` / `calibration_loop_fix.py` 의 `PARAM_DEFAULTS` 는 아직 **기본 MAZE_* 값**일 수 있음.  
실험실 테이블(색지 하드보드 1T, 싸구려 말랑 쿠션)용 **권장 러프 시드**:

| 파라미터 | 권장 시드 | 시뮬 반영 | 실기 반영 |
|----------|-----------|-----------|-----------|
| `ball_restitution` | 0.80 | fast sim·headless eval | 캘리브 최적화 목표 |
| `cushion_restitution` | **0.58** (말랑) | 쿠션 box restitution | 1쿠션 거리 맞춤 |
| `rolling_friction` | 0.022 | 공 rollingFriction | 멈춤 거리 |
| `lateral_friction` | 0.22 | 테이블·공 lateral | 슬라이드 감쇠 |
| `speed_gain_scale` | 1.00 | strike 속도 스케일 | 로봇→공 전달 |
| `table_y_offset` | Phase 1 후 | 비전 y 보정 | 좌표 정합 |

**캘리브 루프:** 실기 trial (카메라 전/후) → headless sim에서 파라미터 튜닝 → `calibration_result_physics.npz` 저장.

## 5. 시뮬 vs 실기 — 역할 분리

```
                    ┌─────────────────────────────────────┐
                    │     CushionShotPlanner (탐색)        │
                    │  + pocket_avoidance (기하 필터)      │
                    └──────────────┬──────────────────────┘
                                   │ strike_dir, speed
           ┌───────────────────────┼───────────────────────┐
           ▼                       ▼                       ▼
   PyBullet GUI/DIRECT      Headless fast sim        Indy 실기
   로봇·공·쿠션 물리         공 속도 grid (로봇 없음)   movel/movej
   홀 없음                   홀 없음                   실물 홀 있음
           │                       │                       │
           └───────────────────────┴───────────────────────┘
                                   │
                          RealSense (공 위치)
```

| 구분 | 시뮬레이션 | 현실 |
|------|------------|------|
| **경로 계획** | 각도 grid + PyBullet 점수 + **홀 회피 필터** | **동일 플래너 출력** 사용 |
| **로봇 실행** | `MoveRobot(q)` / PD | `movel` TCP 직선 |
| **홀** | 없음 (필터만) | 있음 (물리) |
| **공 위치** | 비전 또는 수동 setup | 비전 |
| **물리 튜닝** | config + 캘리브 npz | 실측 trial로 sim 파라미터 맞춤 |
| **좌표** | `calibration_position_offset.json` | Phase 1 수동 보정 |

## 8. 아직 시뮬에 없는 것 (한계)

1. **홀 geometry** — PyBullet 쿠션에 구멍·낙하 없음  
2. **말랑 쿠션 변형** — restitution 숫자 하나로만 근사  
3. **Indy movel in sim** — 시뮬은 관절 공간; 실기는 task space  
4. **teleop→IDLE 대기** — `calibration_loop_fix` 연결부에 미포함 (수동 Conty 확인)
